## 직무 교육
* docker compose
    * 멀티 노드 미션에서 chatgpt가 docker compose를 사용하라고 알려줬었지만, 아직 dockerfile을 작성하는 법도 익숙치 않고, 연습이 더 필요한 것 같다는 생각이 들어 사용하지 않았었음
    * 수정사항 있을 때 다시 빌드하고 재시작하는게 너무 불편했음
    * 그래도 docker 사용법은 좀 익숙해진 것 같고, 다음 미션 때 docker compose 사용해보고자 함
* hadoop 유저를 root로 하지 말자
    * 미션할 때 이런 디테일에 대한 생각이 부족했던 것 같음
    * 주어진 조건만 생각하지 말고, 선택지가 무엇이 있을지에 대한 고민부터 필요한 것 같음

# 학습 기계
## 잘 지키고 있음
1. 매주 금요일 퇴근 전 10분 동안 일주일 동안 쓴 회고를 다시 읽고 리뷰하기
    - 잘 못 지킨 습관은 실천 방법 수정하기
2. 매주 금요일 피드백 받은 내용 최소 1가지 적용해보기

## 노력 필요
1. 모든 작업 최대 30분씩 타임박싱 하고, 연장은 최대 1번만 하기 (작업 = 해결 할 작은 문제 1개 e.g., 도커 파일 작성)
2. 데드라인보다 하루 더 일찍 끝내는 것을 목표로 스케줄링 하기
3. 미션 1개 해결 이후 10분 동안 해당 미션 리뷰하기

## 하면 좋음
1. 집중력이 깨지면 1차 스트레칭, 그래도 집중이 안되면 양치해보기
2. 팀원들과 팀미션 시간에 개인 미션 구현 방식을 3분씩 설명하고 리뷰하는 시간 갖기
3. 과제에 선택사항이 있는 경우 어떤 방법이 더 적합한지 챗지피티에 검색해보고 선택하기
