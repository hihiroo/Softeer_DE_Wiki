# 리뷰
## W3M3 ~ W3M6
* MapReduce 실습을 진행하며 전체 데이터 처리 흐름을 직접 경험함
    * Docker 컨테이너로 로컬 파일을 전달하고
    * 해당 파일을 HDFS에 업로드한 뒤
    * Hadoop에서 MapReduce 작업을 실행하고
    * 실행 결과를 HDFS에서 가져오는 과정을 처음부터 끝까지 직접 수행해봄
* Hadoop Streaming을 사용해 MapReduce 작업을 실행함
    * Mapper가 표준입출력으로 결과를 출력하고, 해당 출력이 Hadoop Job의 입력으로 다시 전달되는 구조임을 이해함
    * 언어 상관없이 MapReduce를 구현할 수 있다는 점에서 매우 유용하다고 느꼈음
    * 이 방식을 활용하면 Hadoop 기반의 실시간 데이터 파이프라인도 구성할 수 있을 것 같아, 이후에 더 깊게 공부해보고 싶다는 생각이 들었음

## W3M2b (검증 단계)
* MapReduce 및 YARN 설정 검증 코드를 작성하는 과정에서 getconf가 호출 오류가 발생
    * 어제 밤에 집에서 검증 코드를 작성하다가 해당 오류 해결에 2시간 이상을 소요함
    * 오늘 다시 약 30분 정도 문제를 살펴봤지만, 타임박싱 시간이 끝나서 다음 30분 안에 해결하기는 어렵겠다고 판단하여 다음 미션으로 넘어감
    * 결과적으로 W3M2b를 7시까지 완전히 마무리하지 못해 아쉬움이 남음
        * 오늘 추가로 더 작업할 예정
* 월, 화 일정이 밀리면서 오늘 스케줄을 촉박하게 잡은 탓에, 검증 코드와 문서 작성을 꼼꼼하게 하지 못한 점이 특히 아쉬웠음
    * 앞으로는 작업 마무리와 검증 시간을 더 여유 있게 확보해야겠다고 느꼈음

## 팀 미션
* 오전에 팀 위키 작성을 진행함
    * YARN 스케줄러의 minimum_allocation_mb 값이 작을 때와 클 때 어떤 차이가 생기는지 팀원들과 함께 토의함
    * 이 과정에서 YARN 스케줄러가 컨테이너에 메모리를 할당하는 방식과 YARN 컨테이너의 개념에 대해 추가로 조사하게 됨
    * 서로 이해가 안 되는 부분과 각자의 생각을 공유하며 토론하는 과정에서 좀 더 깊은 부분까지 생각하게 된 것 같음
