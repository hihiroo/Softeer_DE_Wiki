# 리뷰
## W2M5 팀미션 피드백
* **시나리오 피드백**
    * 시나리오 쓸 때 상황, 누구의, 어떤 문제, 어떻게 해결 포맷 적극 활용하기
    * 타겟 범위가 너무 넓음
        * 프랜차이즈도 최근 급성장 중인 기업인지, 김밥천국 처럼 스테디한 기업인지에 따라 문제와 데이터, 접근 방식이 달라짐
        * 예를 들어, 최근 급성장한 기업이면 서비스는 크게 중요하지 않음
    * 매장마다 편차가 큰 것은 문제가 아님
        * 기대를 충족하지 못하는 매장이 있는 것이 문제
        * 그렇다면 기대 수준을 어떻게 설정해야 할지도 고민해야 함
    * 의사결정을 빠르게 내리기 어렵다 -> 추상적임
        * 가맹점이 당장 수정할 수 있는 문제점을 찾기 어려움
        * Actionable Data를 생각하면서 구체화하기
* **분석 관련 피드백**
    * 부정 word cloud에서 '친절'과 같은 키워드가 있는 건 부정적인 느낌이 안 듦
        * 동의어 맵을 만들어서 '불친절' 같은 키워드에 같이 묶이도록 하면 좋음
    * '위생' 키워드가 나와서 해당 키워드가 언급된 리뷰만 모아서 확인해봤다. 로 끝나면 안됨
        * 그 위생이 매장 관련 위생인지, 사장님이나 직원의 위생인지에 따라 이어질 액션이 다름
        * 결과에 따라 취할 액션이 보일 때까지 더 분석해야 함

### 문제 정의 수정
* **상황**: 
    * 어떤 프렌차이즈 브랜드가 유튜브, SNS를 통해 갑작스러운 히트를 쳐서 단기간에 가맹점 수가 급격히 증가함
    * "생각보다 맛없다", "유튜브에서 본 거랑 다르다", "지점마다 맛이 다르다" 라는 부정적인 반응이 확산된 상황

* **누구의**: 
    * 최근 몇 달 내 가맹점 수가 폭발적으로 증가한 프랜차이즈의 가맹 관리팀 
* **어떤 문제**:
    1. 리뷰에서 언급되는 맛에 관한 문제점이 전체 브랜드의 문제인지, 일부 지점의 문제인지 판단하기 어려움
    2. 각 지점마다 당장 해결할 수 있는 문제점 top3를 찾기 어려움

# 회고

## Keep
* 매주 금요일 직무 교육 끝나고 피드백 받은 내용 1시간 동안 적용해보기
    * 피드백 받은 직후에 다시 보니까 문제점이 더 잘 보이고, 집중도 더 잘되는 것 같음
* 매주 금요일 퇴근 전 10분 동안 일주일 동안 쓴 회고를 다시 읽고 점검하기
    * 왜 이런 회고를 썼었는지 리마인드 되었고, 실제 적용하는데 어떤 어려움이 있었는지를 생각해보면서 좀 더 실천하기 좋은 방법으로 개선하게 됨

## Problem
* 이전 회고에 작성했던 (리뷰를 퇴근 전에 몰아서 정리하지 말고 그때그때 정리하기 + 회고 작성에 30분 이상 투자하지 말기)가 잘 지켜지지 않음
    * 미션 해결에 급급해서 중간 중간 정리하기가 쉽지 않음
    * 미션 1개 해결 이후 10분 동안 해당 미션 리뷰하기
