## 아이디어 구상 (오후)
* 오후에는 오전에 나온 아이디어 중 2개를 선정하고 문제 정의를 좀 더 다듬어서 강사님께 피드백 요청 드림

* 다른 팀과 서로 피드백 요청드린 아이디어를 공유하며 조언을 얻음
    * "직접 찾아보거나 알바를 고용하면 되는 문제 아닌가?"라는 반론이 나왔고, 다시 생각해보니 진짜 문제가 아니라고 생각되었음
    * 타겟을 폐차장이 아니라 폐차하려는 고객으로 바꾸면 문제가 될 수도 있다는 조언을 받음
    * 팀에서 논의해보니 방향이 괜찮은 것 같아 내일 이 문제가 진짜 존재하는지 검증 + 디벨롭하기로 함
    * 이후 강사님께 비슷한 내용의 피드백을 받고 다른 팀에게 조언을 구하는 방법도 정말 도움이 많이 된다는 생각을 함

* 어제 전기차 충전소 관련해서 피드백 받은 내용을 바탕으로 팀원들과 방향성 회의
    * 기업이 아닌 충전소를 이용하는 개인의 고통도 생각해보기로 함
    * 기업의 고통을 어떻게 정량적으로 파악할 수 있을지, 실제 전기차 충전을 중요하게 생각하고 있는지에 대한 자료 조사를 진행하기로 함
    * 결론적으로 자료를 찾지 못했고, 개인의 고통도 충전기 상태를 알려주는 것 만으로는 해결이 안된다고 판단함 -> 아이디어 KILL로 의견을 모음
* notion에 지금까지 나왔던 아이디어별로 **의사결정 로그**를 만들어서 보류, KILL 하는 과정까지의 의견을 기록함
    * 로그를 기록하면서 다시 살릴만한 포인트가 있는지도 검토함
    * 이전에 kill 했던 블랙박스 아이디어에 대해 "타겟이 사고를 즉시 인지하지 못하는 경우가 많다면 kill하기로 했었던 문제점이 사라지는게 아닌가?" 라는 의견이 나왔고 내일 다시 검토해보기로 함
    * 똑같은 아이디어인데 시간이 지난 후에 보니까 생각하지 못했던 부분이 보이는 것 같음 -> 기록이 중요하다!!

* 오후 아이디어 스프린트를 진행하며 "현대자동차가 미래에 이 사업을 진행한다면 어떤 문제를 겪을까?"라는 가상 시나리오 기반 아이디어 정의를 시도함
    * 문제가 너무 넓은 것 같다는 생각이 들었지만 아직 사업이 시작조차 안 된 상황을 전제로 해서 그런지 작은 문제 하나로 좁히기가 쉽지 않았음
        * 자율주행 트럭 운송 사업을 도입 -> 해결해야 할 문제가 너무 많음! -> 가장 작은 문제 하나로 쪼개면 이게 진짜 필요한지 설득력이 떨어지는 것 같음 -> 설득이 안되면 그게 진짜 문제일까? -> 사업 시작하려면 필요하긴 해... -> 무한반복
    * 결과적으로 문제 정의가 잘 안돼 누구의, 어떤 문제, 어떻게 해결을 반복 수정하다가 8시 넘어서 끝남
* 강사님께 피드백 요청드리고 회고를 작성하려다가 팀원들 모두 배고픔 + 피곤함으로 집에 가서 작성하기로 하고 귀가함
    * 어제 회고에서 "남은 일을 집으로 가져가지 않겠다"고 했는데 하루만에 깨버려서 반성 중
    * 의지가 약했다기보다는 시간 배분/타임박싱을 잘못했다고 판단
    * "피드백 받기"처럼 오늘 꼭 이뤄야 하는 목표가 있을 때는 타임박싱을 더 짧게 잡고, 시간 배분 관련 대화를 팀원들과 더 자주, 적극적으로 해야 할 것 같다는 생각을 함
