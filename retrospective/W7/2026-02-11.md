# 리뷰
### 현직자 멘토링
* 우리 아이디어와 데이터 수집 방법, 데이터 크기, 간단한 아키텍처를 설명드림
* 현재 우리 아키텍처가 DE적인 요소가 빠져있고, 백엔드에서 다룰 만한 요소만 들어있다고 피드백 주심
    * 단순히 EC2 / Lambda / S3 / RDS 사용 여부보다 DE 관점의 차별성이 보여야 한다고 말씀하심
        * Spark 최적화, 오류 대응 방식, 정규화 수준, S3 레이어 설계 등
    * 현재 설계는 '무엇을 만들 것인가' 기획 중심이고, '어떻게 안정적이고 확장 가능하게 운영할 것인가'에 대한 설명이 부족하다고 피드백 주심

### 팀 의사결정
* 팀 내 논의 결과 전체 파이프라인을 먼저 빠르게 구현하고 DE 적인 차별점을 덧붙여서 확장하기로 함
* 데이터는 최소 단위로 구축 -> 나중에 확장
    * 영업시간 같은 크게 중요하지 않은 요소도 나중에 추가하기로 함
    * ETL 코드가 정상 동작하는지 확인 가능한 수준만 먼저 구현
    * 이후 데이터 확장 + 최적화 + 에러 대응

### ETL 코드 구현
* 크롤링, Spark Transform으로 역할 나눠서 구현 시작
* 예상보다 구현 시간이 오래 걸릴 것 같아서 마음이 급해짐
* 원래 오늘 구현 + 새로운 아이디어 디벨롭 투 트랙으로 진행하려고 했는데, 구현 중에 이슈가 계속 생겨서 서로 공유하고 해결책 논의에 집중하면서 못 지킴
    * 머리로는 실패 대비용 아이디어를 병행해야 한다고 생각하지만, 마음이 급해지니까 미루게 되는 것 같음
    * aws에 전체 파이프라인 올려보는게 우선인 건 맞지만, 틈 날 때마다(점심시간, 출퇴근 시간, 구현하다 힘들 때) 새로운 아이디어도 계속 구체화하자!
