# 리뷰
### 직무 교육
* **학습 기계를 어떻게 개선할 것인가**
    * 실패했을 때 학습 기계의 문제인지 아닌지를 구분해야 함
    * 회고를 할 때는 구체적으로 실천을 어떻게 할 건지 적어야 함
        * e.g., api를 사용하기 전에 공식문서를 **5분씩** 읽기
* **문제 상황 정의**
    * 가정할 수 있는 케이스가 굉장히 많기 때문에 그 중에서 접근 방식이 매우 크게 달라지는 케이스를 골라서 가정하면 좋음

## 팀 리뷰 (2팀)
### W1M1 - mtcars 데이터셋 분석
* **잘한 점**: 문제 정의 시, 구체적인 비즈니스 상황을 가정하고 접근한 것
* **아쉬운 점**: 너무 하이레벨에서의 답변만 생각함. '실제 비즈니스 상황에서 고객에게 레포트를 제출한다면 어떻게 할까'라는 생각으로 데이터 분석 방향까지 구체적으로 생각했다면 좋았을 것 같음. 

### W1M3 - ETL 프로세스 구현
* 과거 데이터 조회도 더 다양한 시나리오를 생각해봤다면 좋았을 것
    1. **특정 시점별 전체 데이터 조회 필요** (e.g., 2025년 4월 국가별 GDP)
        *  빠른 조회를 위해 시점별 view를 생성하거나 파티셔닝을 적용
    2. **국가별 과거 데이터 조회 필요** (e.g., 대한민국의 전체 GDP 데이터)
        * 국가별 인덱싱을 적용

## Problem
* KPT 회고 작성 시 Problem과 Try를 너무 추상적으로 써왔음
    * 실천 부담이 높아 잘 안 지켜질 확률이 높음

    * 잘 못 지킨 Try, Problem은 실천 방법 점검하고 수정하기
