# 리뷰
### 직무 교육
* **코드 관련**
    * ETL 코드 짤 때는 E, T, L 각 코드를 다른 시스템으로 옮겼을 때도 잘 돌아가게끔 짜야 함
    * URL 주소 같은 configuration은 코드에 넣지 말고 yaml 같은 파일로 따로 빼는 것이 좋음
    * 크롤링 같이 이후에 코드 수정이 필요해질 수도 있는 부분은 함수화 해놓은 것이 좋음
    * 로그 파일은 같은 폴더 말고 로그 폴더 아래에 생성하도록 하기
    * 오류가 있을 때 프로그램이 정상적으로 끝나면 안되고 사용자도 분명히 알 수 있게끔 해야 함 
        * e.g., 오류 raise 하고 프로그램 강제 종료
    * 변수명을 정할 때는 오해가 없도록 해야 함 
        * e.g., 경로인데 변수명에 path가 없으면 파일인지 경로인지 헷갈릴 수 있음
* **학습 관련**
    * 금요일 직무 교육 이후 남는 시간에 피드백 받은 내용 브랜치 따로 파서 적용해보면 도움이 많이 될 것
    
### W2M5
1. 챗지피티에 빅데이터 기반 비즈니스 아이디어 성공 사례 검색
    * 쇼핑몰에서 검색 키워드 및 매출 변화 등의 분석 지표 활용
2. 비슷한 아이디어를 적용할 수 있으면서 데이터 확보에 큰 어려움이 없을 것 같은 비즈니스 분야 탐색
    * 쇼핑몰과 유사한 온라인 사업 구조를 가진 배달앱에 아이디어 적용
3. 같은 아이디어의 사업이 있는지 확인 및 수요 확인
    * 배달의 민족에서 매년 인기 검색어, 인기 메뉴 같은 지표를 제공하고 있지만 실시간 확인은 안됨
    * 배달의 민족 사장님 앱에서도 경쟁 업체 관련 지표는 얻을 수 없었음
4. 데이터 확보 방법 탐색
    * 크롤링 가능 여부 확인
    * 크롤링으로 얻을 수 있는 원천 데이터 리스트업
    * 원천 데이터에서 얻을 수 있는 정보 리스트업
5. 제공 가능한 서비스 선정
6. 배달앱 광고비 조사 및 구독료 선정
