# 리뷰
## 아이디어 회의
* 기존 '독소 조항 분석' 아이디어 폐기함
    * 업계를 잘 몰라서 타겟 설정과 문제 정의가 좀 억지스럽다는 느낌이 있었음
    * '이게 독소조항입니다'를 알려줘도 고객이 취할 수 있는 다음 행동이 바로 이어지지 않아서 '변호사를 대체할 수 있는가?'라는 질문을 해결 못함
    * 다노님 피드백을 받고 팀원들과 논의한 결과 무엇이 진짜 문제인지조차 명확히 설명 못하는 상태라는 걸 자각
* 전략을 전환하여 '우리가 잘 아는 문제'에서 찾아보기로 함
    * 도메인 이해 없이 문제 정의하는 것에 한계를 느낌
    * 오후 내내 30분 단위 아이디어 스프린트 진행
        * 저번주 아이디어 생각만 하다가 일정 딜레이 되었던 경험을 되풀이 하지 않기 위해 짧게 설정
        1. 아이디어 제안
        2. 이게 진짜 문제인가?
        3. 해결 가능한가?
        4. 100만원의 가치가 있는가?  
        를 팀원들과 하나씩 합의
    * 생각이 안나면 '난임 부부', '햇빛 알레르기가 있는 친구' 와 같이 지인이나 내가 겪었던 경험과 관련된 키워드를 나열하고 이 중에 해결 가능한 문제가 있을지 함께 고민
* 아이디어 선정 및 구체화
    * 반려동물 보호자로서 느꼈던 나의 고통을 기반으로 한 수의사 추천 서비스로 방향 확정
    * 고객 입장에서 어떤 데이터를 필요로 할지, 동물병원에 갔을 때 이 서비스를 고민할 만한 타겟이 누굴지 경험을 기반으로 정의
* 프리토타입 제작 및 아이디어 점검
    * 웹 페이지 프리토타입을 제작하여 실제 고객이 이 페이지를 봤을 떄 지불 의사가 생길지 확인

# 학습 기계
## 잘 지키고 있음
1. 매주 금요일 퇴근 전 10분 동안 일주일 동안 쓴 회고를 다시 읽고 리뷰하기
    - 잘 못 지킨 습관은 실천 방법 수정하기
2. 매주 금요일 피드백 받은 내용 최소 1가지 적용해보기

## 노력 필요
1. 모든 작업 최대 30분씩 타임박싱 하고, 연장은 최대 1번만 하기 (작업 = 해결 할 작은 문제 1개 e.g., 도커 파일 작성)
2. 데드라인보다 하루 더 일찍 끝내는 것을 목표로 스케줄링 하기
3. 미션 1개 해결 이후 10분 동안 해당 미션 리뷰하기
